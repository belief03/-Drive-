# ダウンロードが Chrome でブロックされる場合

CSVエクスポートや領収書PDFなど、本ツールからのダウンロードが Google Chrome で「危険なダウンロード」としてブロックされることがあります。特に **HTTP（暗号化されていない）でローカル実行しているとき** に起こりやすいです。

## 対処方法

### 1. ダウンロードバーから「保持する」を選ぶ

Chrome の画面下部に「〇〇.csv は危険なためブロックされました」と出た場合：

1. ダウンロードバー右側の **▼** をクリック
2. **「保持する」**（または「保存」）を選択

信頼しているローカル環境（自分で動かしている帳簿ツール）のファイルであれば、この操作で保存できます。

### 2. 設定でブロックを緩和する（任意）

- Chrome の **設定** → **プライバシーとセキュリティ** → **セキュリティ**
- **「セーフ ブラウジング」** のレベルを変更できる場合があります（環境により表示は異なります）

**注意:** 不明なサイトからのダウンロードはブロックしたままにし、**自分が実行している localhost / 127.0.0.1 のツール**だけ許可するようにしてください。

### 3. HTTPS で運用する

本番や共有環境で使う場合は、HTTPS（SSL）でサーバーを動かすと、Chrome がブロックしにくくなります。ローカル開発では HTTP のままで、「保持する」で保存する運用で問題ありません。

## 文字化けする場合

### CSV を Excel で開いたときに文字化けする

CSV は **BOM 付き UTF-8** で出力しています。それでも Excel で直接開いて文字化けする場合は、次の方法で開いてください。

1. Excel で **「データ」** → **「テキストまたは CSV から」** を選択
2. ダウンロードした CSV を選ぶ
3. ファイルの元の形式で **「65001: Unicode (UTF-8)」** を選んで読み込む

### 領収書 PDF の日本語が文字化けする

領収書まとめ PDF では、日本語用の CID フォント（HeiseiMin-W3 / HeiseiKakuGo-W5）を使うようにしています。古い環境やフォントが入っていない場合は、reportlab を最新にしておいてください（`pip install -U reportlab`）。

---

## 本ツール側の対応

- ダウンロード時に **Content-Type** と **X-Content-Type-Options: nosniff** を付与し、通常のファイル配信として認識されやすくしています。
- CSV は **BOM 付き UTF-8**（`text/csv; charset=utf-8`）、PDF は **application/pdf** で返しています。PDF の日本語は CID フォントで描画しています。

それでもブロックされる場合は、上記「保持する」で保存してご利用ください。
