# 壁打ち内容・実装記録・次回やれること

---

## 1. 壁打ち内容（設計で決めたこと）

| 項目 | 内容 |
|------|------|
| **利用者** | 個人事業主（のちにスタッフ想定）、簿記は初めて |
| **入口** | 取引タイプで選ぶ（売上・雑収入・経費・振替・借入・事業主貸借・その他） |
| **経費** | リード型で勘定科目を提案。同一取引先で前年と違う科目を選んだらアラート。支払方法にQR決済（引き落とし元を選択） |
| **設計思想** | アシスト重視・AI（システム）リード型。蓄積データは将来の学習・提案にも利用 |
| **業種** | 初期設定で1回選択、設定画面でいつでも変更可。前提はIT、請求書ベースの売上・外注費・通信費・交通費中心 |
| **スコープ** | 月200件想定。日々の入力＋P/L・B/S集計、確定申告用エクスポート。e-Taxは「データ出力」まで（提出は別途） |
| **データ** | 第一期はローカル。バックアップ推奨・7年保存。手入力＋CSV取込を第一期に含める。OCRは後回し |
| **領収書** | 領収書保管済フラグ・メモ。画像添付は推奨だが第一期必須ではない。月別・科目別ファイリング・7年保存を案内 |
| **法改正** | 勘定科目・様式はマスタ化・設定の外在化で変更しやすくする |
| **技術** | Webアプリ（Python）、DBはSQLite |
