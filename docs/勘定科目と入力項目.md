# 勘定科目リストと取引タイプ別入力項目

青色申告用帳簿ツールの設計ドキュメント（壁打ち成果）。

---

## 0. 前提（業種・方向性）

- **業種**：初期ユーザーはIT関係。将来は他業種や事業拡大に備え、**業種を選択できるように実装**する。
- **第一期で想定する主な取引**：売上＝請求書ベースの入金が中心。経費＝外注費・通信費・交通費が中心。
- **業種選択の役割**：選択した業種に応じて、推奨勘定科目セットや「よくある経費」の並び・表示を変えられるようにする（将来の拡張）。事業が多くなった場合も、業種を変更・追加する前提で設計する。

### 1.2 「よくある経費」と「その他」の関係

- **よくある経費**：画面でショートカット表示する科目（ユーザーが選びやすい）。  
  → 選んだら上記の「費用」のいずれかに対応する勘定科目に自動設定する。
- **その他の経費**：上記「費用」のフルリストから選択。

## 3. DB・画面設計への反映メモ

- **勘定科目**：マスタテーブルに「科目コード・勘定科目名・区分（資産/負債/純資産/収益/費用/事業主貸借）」を保持。  
  「よくある経費」は、同じマスタへのショートカット（表示順・フラグ）で管理。
- **取引**：取引タイプ・日付・金額・借方科目・貸方科目・取引先・摘要・支払方法（経費の場合）などを保存。  
  取引先は任意だが、入力されている場合は提案・アラートに使用。
- **CSV取込**：明細1行 → 上記「経費」または「売上」の入力項目にマッピングし、候補として保存。  
  取引先・摘要から勘定科目を提案するロジックは手入力と共通にする。

## 4. 業種選択の仕様（実装方針）

- **初期設定で1回選ぶ**：初回起動時（または初回ログイン時）に業種を選択する。
- **設定画面でいつでも変更可**：状況が変わったタイミングで再度選択できる。設定画面から業種を変更可能とする。
- **業種の選択肢（案）**：IT・Web受託、コンサル、物販、飲食、士業、その他（自由記述も可）。
- **業種に応じる挙動（将来の拡張）**：推奨勘定科目セットの表示順や「よくある経費」の並びを業種ごとに変える。データは引き継ぎ、勘定科目は追加・無効化で対応する。

## 5. 法改正に柔軟な実装方針

法改正に強いシステムとするため、次の方針で実装する。

- **マスタデータ化**：勘定科目一覧・業種リスト・取引タイプ・申告書で使う項目などは、コードに直書きせずDBまたは設定ファイル（JSON/YAML等）で管理する。法改正で科目や様式が変わっても、マスタの更新・追加で対応できるようにする。
- **設定の外在化**：税率・控除額・表示ラベル・必須項目など、法改正で変わりうる値は設定として切り出し、コードの修正なしで変更できるようにする。
- **バージョン・有効期間**：勘定科目や申告様式に「有効開始日・有効終了日」やバージョンを持たせ、過去年度の帳簿は当時のルールで参照できるようにする（将来拡張）。
- **変更履歴**：マスタや設定の変更履歴を残せるようにし、いつ何を変えたかを追えるようにする（運用・監査用）。

## 6. 領収書・レシートの保管ととりまとめ

### 6.1 税務上の前提

- 経費の計上には **証拠書類（領収書・レシート等）の保存** が求められる。税務調査で提示を求められることがある。
- **保存期間**：原則 **7年**（前々年分までは7年、それ以前は5年など例外あり。ここでは7年を目安とする）。
- 紙のままでも、**電子で保存**してもよい。電子の場合は電子帳簿保存法の要件（タイムスタンプ等）に注意。ツールで「画像を添付して保管」する場合は、公式な電子保存要件を満たすかは別途確認が必要（ツール内での「とりまとめ・照合のしやすさ」が主目的とする）。

### 6.2 とりまとめの考え方

- **整理の単位**：月別、または **月別＋勘定科目別** でファイリングする方法が一般的。確定申告や調査のときに「○年○月の通信費」のように探しやすい。
- **ツールとの対応**：ツールで「月別・科目別の経費一覧」を出せるようにしておくと、**帳簿の経費一覧** と **手元の領収書のとりまとめ** を対応づけやすい。例：経費一覧を印刷またはCSVで保存し、同じ順で領収書をファイルに綴じる。

### 6.3 ツールでどこまで扱うか（方針）

| 機能 | 第一期 | 備考 |
|------|--------|------|
| **仕訳に「領収書あり」のメモ** | 対応する | 任意のメモ欄や「領収書保管済」フラグで、「この経費は領収書を保管している」と記録できるようにする。保管場所（例：自宅・月別ファイル）をメモしてもよい。 |
| **領収書・レシート画像の添付** | 任意で対応可 | 経費の仕訳に画像ファイルを1件ずつ紐づけて保存する。とりまとめは「ツール上で経費一覧を見ると、その行に領収書画像が付いている」形で一元管理できる。保存先はDBにバイナリを格納するか、フォルダに保存してパスをDBに持つかは実装で選択。 |
| **OCR（画像から日付・金額を読み取り）** | 後回し | 領収書画像から文字認識で仕訳候補を作る機能は、将来の拡張とする。 |
| **レポートでの「領収書あり」表示** | 対応する | 経費一覧や月次集計で「領収書あり／未」やメモを表示できるようにし、とりまとめ漏れに気づきやすくする。 |

- **推奨**：第一期では **「領収書保管済」フラグまたはメモ欄** を必ず用意し、**画像添付** は「あると便利」な程度で実装するかどうか決める。とりまとめのしやすさのため、**月別・科目別の経費一覧の出力** は実装する。

### 6.4 運用上のヘルプ・推奨

- 画面やヘルプに「領収書・レシートは **月別・科目別** にファイリングし、**7年保存** を推奨」と短く記載する。
- 電子で保存する場合の注意（電子帳簿保存法、タイムスタンプ等）は、ヘルプまたはリンクで案内できるようにしておく。
