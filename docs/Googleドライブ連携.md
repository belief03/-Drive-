# Googleドライブと本システムのAPI連携

## 1. 連携の目的

| 用途 | 説明 | 優先度 |
|------|------|--------|
| **バックアップ** | 指定フォルダにDB・CSV・領収書PDFを定期的または手動でアップロードし、7年保存の一環としてドライブに保管する | 高 |
| **領収書の二重保管** | 登録した領収書画像をドライブのフォルダにも保存し、ローカル障害時や複数端末から参照しやすくする | 中 |
| **ドライブ上のCSV取込** | 銀行・クレジットの明細CSVをドライブに置き、本システムから「ドライブのファイルを選択して取込」できるようにする（**実装済**） | 中 |

## 2. 必要な準備（Google Cloud側）

**注意**: 単純な「APIキー」ではなく、**OAuth 2.0 の認証情報（クライアントID・クライアントシークレット）** を作成します。ユーザーが「このアプリにドライブへの保存を許可する」と認可するために必要です。

### 認証情報の作成手順

1. **Google Cloud Console** を開く  
   https://console.cloud.google.com/

2. **プロジェクトを作成**  
   - 画面上部のプロジェクト選択 → 「新しいプロジェクト」→ 名前（例: 帳簿ツール）を入力して作成。

3. **API を有効化**  
   - **Google Drive API**: 「APIとサービス」→「ライブラリ」→「Google Drive API」を検索 → 「有効にする」。  
   - **Google Sheets API**（取引一覧をスプレッドシートで共有する場合）: 同じ「ライブラリ」で「Google Sheets API」を検索 → 「有効にする」。

4. **OAuth 同意画面を設定**（初回のみ）  
   - 「APIとサービス」→「OAuth 同意画面」  
   - **ユーザータイプ**  
     - **外部** … 個人の Google アカウント（@gmail.com など）や任意の Google アカウントで利用可能。**自分用・個人事業主向けの本ツールでは「外部」を選びます。** テスト公開のままにすれば、自分を「テストユーザー」に追加するだけで審査なしで利用できます。  
     - **内部** … Google Workspace の組織内ユーザーのみ。組織の管理者が作る業務用アプリ向け。個人の @gmail.com では利用できません。  
   - アプリ名・ユーザーサポートメールなどを入力して保存。  
   - 「スコープを追加」→ `https://www.googleapis.com/auth/drive.file` を追加して保存。  
   - 「外部」を選んだ場合、後から「テストユーザー」に自分のメールアドレスを追加すると、公開審査なしで認可できます。

5. **認証情報（OAuth 2.0 クライアントID）を作成**  
   - 「APIとサービス」→「認証情報」→「認証情報を作成」→「**OAuth クライアント ID**」を選択。  
   - アプリケーションの種類: **「ウェブアプリケーション」**（本システムはブラウザで認可するため）。  
   - **名前**: コンソール上で自分が分かるラベルでよい。例は次のいずれかでOK。  
     - `帳簿ツール ローカル` … ローカルで動かす用と分かる  
     - `青色申告 帳簿ツール` … アプリ名に合わせる  
     - `あおいろサポート Drive連携` … プロジェクト名に合わせる  
     ※ この名前はユーザーの認可画面には出ません。Google Cloud の認証情報一覧で識別するためのものです。  
   - **認可済みのリダイレクト URI** に以下を追加（複数登録可。どのアドレスで開いても連携できるようにする）:  
     - `http://127.0.0.1:5000/settings/drive/callback`  
     - `http://localhost:5000/settings/drive/callback`  
     - （同じPCのIPで開く場合）例: `http://172.20.10.2:5000/settings/drive/callback`  
   - 「作成」をクリック。

6. **クライアントIDとシークレットを控える**  
   - 表示された **クライアント ID** と **クライアント シークレット** をコピーし、環境変数に設定する（下記「7. 環境変数」参照）。  
   - シークレットは他人に共有・リポジトリにコミットしないこと。

## 3. 技術構成

- **API**: Google Drive API v3
- **認証**: OAuth 2.0（認可コードフロー）。リフレッシュトークンをDB（Setting）または `instance` 内の暗号化ファイルに保存し、再認可を減らす。
- **Python ライブラリ**: `google-auth`, `google-auth-oauthlib`, `google-api-python-client`
- **スコープ例**:
  - `https://www.googleapis.com/auth/drive.file` … アプリが作成したファイルのみ（推奨）
  - `https://www.googleapis.com/auth/drive.readonly` … ユーザー指定フォルダの読み取り（CSV・画像取込用）
  - `https://www.googleapis.com/auth/spreadsheets` … スプレッドシートの作成・編集（取引一覧の「スプレッドシートに共有」用）

## 4. 本システム側の実装方針

- **設定画面** に「Googleドライブ連携」を追加する。
  - 「連携する」で OAuth フローを開始し、取得したリフレッシュトークンを安全に保存する。
  - 「連携を解除する」でトークンを削除する。
- **バックアップ**  
  - 手動: 「バックアップをドライブに保存」ボタンで、その時点の DB コピー（または CSV エクスポート）と任意で領収書まとめPDFを、ドライブの指定フォルダ（例: `帳簿バックアップ/YYYY-MM-DD/`）にアップロードする。
  - 自動: オプションで「毎月1回、ドライブにバックアップ」など定期実行（スクリプト or アプリ内スケジューラ）。
- **領収書の二重保管**  
  - 領収書画像アップロード時に、連携済みなら同じファイルをドライブのフォルダ（例: `帳簿_領収書/`）にもアップロードし、Transaction に `drive_file_id` を保存する（任意。まずはバックアップで「領収書まとめPDF」を保存する形でも可）。
- **ドライブ上のCSV取込**（実装済）  
  - 取込画面の「GoogleドライブのフォルダからCSVを選択して取込」から、フォルダIDを入力してファイル一覧を表示し、CSVを選ぶと本システムに取込める。スコープに `drive.readonly` を追加しているため、**既に連携済みの場合は一度「連携を解除」→「連携する」をやり直す**と、ドライブのフォルダを参照できるようになる。

## 5. セキュリティ・注意点

- クライアントシークレットは環境変数や `instance` 内の設定に置き、リポジトリにコミットしない。
- リフレッシュトークンは暗号化して保存するか、アクセス制限された場所に置く。
- 本システムは第一期はローカル運用を想定しているため、OAuth のリダイレクトURI は `http://127.0.0.1:5000/...` などでよい。本番で HTTPS にする場合はリダイレクトURI を追加登録する。
- 「OAuth 2 MUST utilize https」というエラーは、OAuth ライブラリが HTTP を拒否しているため。アプリ側でローカル用に `OAUTHLIB_INSECURE_TRANSPORT=1` を設定しているため、通常は解消される。
- 利用するスコープは必要最小限（`drive.file` を優先）とし、ユーザーに「どのフォルダに保存するか」を選ばせる場合はフォルダIDだけ保存する。

## 6. 実装ステップ

1. **Phase A**（実装済）: 設定に「Googleドライブ連携」を追加。OAuth フローとリフレッシュトークン保存。連携状態の表示・解除。
2. **Phase B**（実装済）: 「バックアップをドライブに保存」で、指定年の取引CSV と領収書まとめPDF をドライブの「帳簿バックアップ」フォルダにアップロードする。
3. **Phase C**（一部実装済）: ドライブ上のCSVを選択して取込 → 実装済。領収書画像のドライブ二重保存は任意。
4. **取引一覧をスプレッドシートで共有**（実装済）: 取引一覧画面の「スプレッドシートに共有」で、表示中の取引（期間絞り込みありの場合はその範囲）を新規の Google スプレッドシートに書き出し、共有用のURLを表示します。Google Sheets API を有効化し、既に連携済みの場合は必要に応じて「連携を解除」→「連携する」でスプレッドシート用スコープを追加してください。

## 7. クライアントID・シークレットの渡し方

**「Googleドライブと連携する」ボタンが出ない**場合は、クライアントIDとシークレットがアプリに渡っていません。次のどちらかで設定してください。

### 方法A: 環境変数

| 変数名 | 説明 |
|--------|------|
| `GOOGLE_DRIVE_CLIENT_ID` | Google Cloud の OAuth 2.0 クライアントID |
| `GOOGLE_DRIVE_CLIENT_SECRET` | クライアントシークレット |
| `GOOGLE_DRIVE_REDIRECT_URI` | （任意）未設定時は `http://127.0.0.1:5000/settings/drive/callback` |

※ 環境変数は **アプリを起動したターミナル** で設定したうえで `python app.py` を実行してください。

### 方法B: ファイルで設定（環境変数が難しい場合）

プロジェクト内の **`instance`** フォルダに、次の2つのテキストファイルを作成し、**1行目だけ**に値を書いて保存します。

- **`instance/google_drive_client_id.txt`** … 1行目にクライアントID（`xxxxx.apps.googleusercontent.com`）
- **`instance/google_drive_client_secret.txt`** … 1行目にクライアントシークレット（`GOCSPX-xxxxx`）

※ `instance` フォルダはアプリ初回起動で自動作成されます。ない場合は手動で作成してください。  
※ これらのファイルは **git にコミットしない**でください（シークレットが含まれます）。

## 8. Drive のフォルダを本システムで使う（取込）

**ドライブにある特定フォルダ（例: 領収書フォルダに置いたCSV）を本システムに取り込む**ことができます。

### CSV の取込

1. **取引** → **CSV取込** を開く。
2. 「**GoogleドライブのフォルダからCSVを選択して取込**」をクリック。
3. **フォルダID** を入力する。  
   ドライブでフォルダを開いたときのURLの `folders/` の後ろの文字列がフォルダIDです。  
   例: `https://drive.google.com/drive/folders/1ABC...xyz` → フォルダIDは `1ABC...xyz`
4. 「**フォルダ内のファイルを表示**」を押すと、そのフォルダ内のCSV一覧が表示されます。
5. 取込したいCSVの「**このファイルを取込**」をクリック。
6. 列の指定（日付・金額・内容の列）をして「読み込む」→ いつもの取込プレビュー・登録の流れになります。

**注意**: この機能では **ドライブの読み取り権限**（`drive.readonly`）を使います。すでに「連携する」だけを済ませている場合は、一度**設定で「連携を解除」→ 再度「連携する」**をやり直すと、新しい権限が付与され、フォルダ一覧が表示されるようになります。

### 領収書フォルダの画像について

領収書**画像**（PDF・写真）をドライブの「領収書」フォルダから本システムに一括取込する機能は、現時点では未実装です。CSV取込と同様の「フォルダ指定→ファイル選択→取込」を領収書画像用に追加する場合は、別途実装が必要です。

### 取引一覧をスプレッドシートで共有

取引一覧のデータを **新規の Google スプレッドシート** に書き出し、共有用のURLを取得できます。

1. **取引一覧** を開き、必要なら「期間」で絞り込む。
2. **「スプレッドシートに共有」** をクリック。
3. 新しいスプレッドシートが作成され、一覧に「スプレッドシートを開く」リンクが表示される。そのリンクから開き、Google の「共有」で他者に共有できる。

**事前準備**: Google Cloud で **Google Sheets API** を有効化してください（「APIとサービス」→「ライブラリ」→「Google Sheets API」→「有効にする」）。

#### スプレッドシート用スコープを付与する方法（既にドライブ連携済みの場合）

一度「連携する」を済ませていると、**その時点ではスプレッドシート用の権限が含まれていません**。スプレッドシートに共有するには、もう一度認可し直して新しいスコープを追加する必要があります。手順は次のとおりです。

1. 本アプリの **設定** 画面を開く。
2. **「Googleドライブ連携」** のところで **「連携を解除する」** をクリックする。  
   （トークンが削除されるだけで、Google Cloud の設定やドライブのファイルには影響しません。）
3. 同じ画面で **「連携する」** をクリックする。
4. Google の認可画面が開いたら、**同じ Google アカウント** でログインし、表示される権限を確認して **「許可」** を押す。  
   このとき、ドライブに加えて **「Google スプレッドシートの作成・編集」** などのスコープが新たに含まれ、次回から「スプレッドシートに共有」が使えるようになります。
5. 認可が完了すると設定画面に戻る。「連携済み」と表示されていれば、取引一覧の「スプレッドシートに共有」が利用可能です。

## 9. 設定ページで「接続拒否」になる場合

**「設定」タブや設定のURL（例: `/settings/`）を開くと「接続拒否」「ERR_CONNECTION_REFUSED」と表示される**ときは、次を順に確認してください。

1. **アプリが起動しているか**  
   ターミナルで `python app.py` を実行したままにし、`Running on http://0.0.0.0:5000` などの表示が出ているか確認する。終了している場合は再度起動する。

2. **同じアドレスで開いているか**  
   最初に開いたときのアドレス（`http://127.0.0.1:5000` か `http://localhost:5000`）と、設定を開くときのアドレスを**同じ**にする。ブックマークやリンクが別のホスト（例: 片方が 127.0.0.1、もう片方が localhost）になっていると、接続拒否のように見える場合がある。

3. **設定ページでサーバーが落ちていないか**  
   設定を開いた直後にサーバーが止まると、次にどのページを開いても接続拒否になる。ターミナルでエラーが出ていないか確認し、出ている場合はその内容を手がかりに修正する。設定画面側のエラーはできるだけ捕捉するようにしているため、画面に「設定の読み込み中にエラーが発生しました」と表示される場合もある。

4. **ファイアウォールやポート**  
   別のPCやスマホから `http://＜このPCのIP＞:5000` で開いている場合、ファイアウォールで 5000 番が許可されているか、`app.py` で `host="0.0.0.0"` で待ち受けているかを確認する。

### 9.1 IPアドレス（172.20.10.2 など）で「接続が拒否されました」になる場合

**スマホや別のPCから `http://172.20.10.2:5000` のようにIPで開くと「このサイトにアクセスできません」「接続が拒否されました」「ERR_CONNECTION_REFUSED」と出る**ときは、次を確認してください。

1. **アプリを「帳簿を動かしているPC」で起動する**  
   `172.20.10.2` は、Wi‑Fiや有線でつながっている「このPC」のIPです。**そのPCのターミナルで** `python app.py` を実行し、`Running on http://0.0.0.0:5000` と表示されたままにしておく。アプリを止めていると、どの端末から開いても接続拒否になります。

2. **同じPCのブラウザで開くなら 127.0.0.1 を使う**  
   帳簿を動かしている**同じPC**のブラウザで開く場合は、`http://127.0.0.1:5000` または `http://localhost:5000` を使ってください。IP（172.20.10.2）ではなく 127.0.0.1 にすると、ファイアウォールの影響を受けにくくなります。

3. **Windows ファイアウォールでポート 5000 を許可する**（他端末から IP で開く場合）  
   別の端末（スマホ・他のPC）から `http://172.20.10.2:5000` で開くには、**172.20.10.2 のPC** でファイアウォールがポート 5000 の通信を許可している必要があります。  
   - **Windows**: 「Windows Defender ファイアウォール」→「詳細設定」→「受信の規則」→「新しい規則」→「ポート」→ TCP 5000 → 「接続を許可する」→ プロファイルで「プライベート」などにチェック。  
   - または、初回で「Python をファイアウォールで許可しますか？」と出たときに「アクセスを許可」を選ぶ。  
   本アプリは `host="0.0.0.0"` で待ち受けているため、ファイアウォールさえ通れば同じネットワーク内の他端末からアクセスできます。

4. **IP が変わっていないか確認する**  
   Wi‑Fi やルーターの設定で、PCのIP（172.20.10.2）は変わる場合があります。接続拒否になったら、そのPCで「設定」→「ネットワーク」などから現在のIPを確認し、`http://＜表示されたIP＞:5000` で開き直してください。

## 10. 認可に失敗する場合（「連携する」を押しても失敗する）

**「Googleドライブの認可に失敗しました」と出る**場合、次の点を確認してください。

1. **Google Cloud に複数ホストを登録する（推奨）**  
   本アプリは「今開いているURL」に合わせてリダイレクトURIを使うため、**使う可能性のあるアドレスをすべて**「認可済みのリダイレクト URI」に追加しておくと便利です。  
   - `http://127.0.0.1:5000/settings/drive/callback`  
   - `http://localhost:5000/settings/drive/callback`  
   - 同じPCのIPで開く場合（例）: `http://172.20.10.2:5000/settings/drive/callback`  
   設定画面に表示されている「今回使うリダイレクトURI」が Google Cloud に登録されていれば問題ありません。

2. **Google Cloud のリダイレクトURI**  
   「認証情報」→ 該当の OAuth クライアント ID → 「認可済みのリダイレクト URI」に、  
   上記のいずれか（または設定画面に表示されているURI）が **一字一句同じ**で登録されているか確認してください。

3. **クライアントID・シークレット**  
   環境変数または `instance` 内のファイルに、正しい値が入っているか確認してください。  
   アプリを再起動したうえで再度「連携する」を試してください。

4. **ログで原因を確認**  
   アプリをターミナルで起動している場合、失敗時に `state mismatch` や `token exchange failed` などのログが出ます。  
   - `state mismatch` → 上記 1 の「127.0.0.1 で開く」を試してください。**本アプリは localhost で開いた場合、自動で 127.0.0.1 にリダイレクトします。** 一度 127.0.0.1 で開き直してから「連携する」を押してください。  
   - `token exchange failed` → 上記 2・3 のリダイレクトURI・認証情報を確認してください。

### 設定に誤りがなさそうなのに「トークン取得に失敗しました」と出る場合

- **ターミナルで実際のエラーを確認する**  
  コマンドラインで `python app.py` を実行し、そのまま「連携する」→ Google で許可 → 戻ってきたときに表示される **赤いエラー文** を確認してください。Google が返している内容（例: `redirect_uri_mismatch`、`invalid_grant`）がそのまま出ます。
- **リダイレクトURIの登録を1件だけにする**  
  「認可済みのリダイレクト URI」に複数登録している場合、本アプリ用は **`http://127.0.0.1:5000/settings/drive/callback` の1件だけ** にし、前後にスペースや改行が入っていないか確認してください。
- **環境変数と Google Cloud を一致させる**  
  `GOOGLE_DRIVE_REDIRECT_URI` を自分で設定している場合は、Google Cloud に登録した URI と **完全に同じ** にしてください。未設定ならアプリは `http://127.0.0.1:5000/settings/drive/callback` を使います。
- **認可コードは1回だけ**  
  「許可」したあと、戻ってきた画面でリロードや「連携する」の再クリックをすると、認可コードが再利用されて `invalid_grant` になることがあります。その場合は「連携する」から最初からやり直してください。
